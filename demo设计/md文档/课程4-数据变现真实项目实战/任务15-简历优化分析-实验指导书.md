# 任务15：简历优化分析

## 📌 学习目标

通过本任务的学习，你将能够：

1. **知识目标**
   - 理解“简历与 JD 关键词匹配”的重要性  
   - 了解用数据和事实来支撑简历内容的方法（数据量化）  
   - 理解不同岗位需要“不同版本简历”的原因  

2. **技能目标**
   - 能够对照 JD 分析自己简历的关键词覆盖情况  
   - 能够用 STAR + 数字的方式改写经历描述  
   - 能够在 AI 辅助下，针对不同岗位定制优化后的简历版本  

3. **素养目标**
   - 养成从 HR / 用人经理视角看简历的习惯  
   - 建立真实、准确又有说服力的表达意识  
   - 形成“持续迭代简历”的职业习惯，而不是“一份简历用到底”

## 🧭 操作总体思路

本任务的核心问题是：

1. **简历是否说对了“对方想听的话”？**——即，是否包含 JD 中的核心关键词？  
2. **简历是否用“事实和数据”证明了你的能力？**——而不是空泛的“参与、负责、协助”。  
3. **简历是否对目标岗位“有针对性”？**——不同岗位看到的重点是否不一样？  

你会依次经历：  
**对照 JD 找差距 → 给经历加上数字 → 使用 AI 做针对性优化 → 输出 1–2 个版本的简历**。

## 📋 操作步骤

### 步骤一：从 HR 视角理解简历筛选（约15分钟）

#### 1.1 站在 HR 的位置想一想

先在纸上或 Word 中写下你对以下问题的直觉答案：

- 一个热门岗位，一般会收到多少份简历？  
- HR 大概会用多少时间快速“扫一眼”一份简历？  
- 如果你是 HR，你会优先看哪些信息？（例如：专业对不对口、技能匹不匹配、经历是否相关、有无数据证明等）  

然后对照教案中的要点理解：

- HR 在**最开始的几秒钟**，通常只在寻找：  
  - 岗位关键词（Excel / SQL / 运营 / 数据分析 / 行业名等）；  
  - 与岗位高度相关的经历标题或项目名；  
  - 能一眼看出“结果”的数字（增长率、规模、百分比等）。  

> 记住：简历的首要任务是“让 HR 愿意多看几秒”，而不是“一次把人生故事讲完”。

### 步骤二：做简历与 JD 的关键词匹配分析（约25分钟）

目标：找出**JD 要求的关键词中，哪些已出现在你的简历里、哪些还缺失**。

#### 2.1 从 JD 中提取关键词

选定一个具体岗位 JD（建议选你最想投的方向），可以参考任务14 的技能列表，提取以下几类词：

- **技能词**：Excel、SQL、Python、数据可视化、统计分析等  
- **方法词**：数据清洗、建模、用户分析、A/B 测试等  
- **成果词**：优化、提升、增长、降低、转化、留存等  
- **行业/业务词**：电商、短视频、新媒体、SaaS 等  

可以直接让 AI 帮你提取，提示词示例：

```text
“请从下面这份 JD 中提取核心关键词，分为：
1）硬技能（工具/技术/方法）；
2）软技能；
3）行业/业务相关词。
并给出一个关键词列表，方便我逐项检查。”

JD 内容：
[粘贴 JD]
```

整理到一个简单的对照表中：

| 关键词       | 类型     | 是否在 JD 中出现？（默认是） |
|--------------|----------|-------------------------------|
| Excel        | 硬技能   | 是                            |
| SQL          | 硬技能   | 是                            |
| 数据清洗     | 方法     | 是                            |
| 数据可视化   | 硬技能   | 是                            |
| 沟通表达能力 | 软技能   | 是                            |

#### 2.2 检查你的简历中是否覆盖这些关键词

打开你当前的简历，逐条对照刚刚的关键词表，在新的一列标记“简历中是否有”以及“出现的位置”：

| 关键词       | 类型   | 简历中是否有？ | 位置（技能栏/项目1/自我评价等） |
|--------------|--------|----------------|----------------------------------|
| Excel        | 硬技能 | 有             | 技能栏 + 项目1                  |
| SQL          | 硬技能 | 有             | 项目2                            |
| Python       | 硬技能 | 无             | -                                |
| 数据清洗     | 方法   | 有             | 项目1                            |
| 数据可视化   | 硬技能 | 有             | 技能栏                            |
| 统计分析     | 硬技能 | 无             | -                                |

你可以估算一个“关键词覆盖率”：

```text
覆盖率 ≈ （简历中出现的 JD 核心关键词个数） / （JD 核心关键词总数） × 100%
目标：尽量做到 70% 以上。
```

> 注意：不要为了“覆盖率”乱堆你不会的技能，**只能写你真的用过或能在面试中讲清楚的内容**。

### 步骤三：用数据量化你的经历（约25分钟）

很多同学的简历经历描述往往是：  
“参与 XX 项目；负责数据分析；帮助团队做决策。”  
——看完之后，HR 还是不知道你“到底做了什么、做得怎么样”。

本步骤会用 STAR + 数据法则帮你重写。

#### 3.1 选出 2–3 条最重要的经历

从你的简历中，选出你认为**最能代表自己能力**的 2–3 条经历，例如：

- 某个课程项目（如“小店经营分析”“电商选品分析”等）；  
- 实习经历；  
- 校园活动经历（如果与数据/运营相关）。  

把原始描述复制到一个临时文档中备用。

#### 3.2 按 STAR 结构拆分经历

对每条经历，用简单的方式回答四个问题（可直接写成 4 行）：

```text
S（情境 Situation）：当时的大背景是什么？项目目标是什么？
T（任务 Task）：你具体负责的任务是什么？
A（行动 Action）：你做了哪些关键动作？用了哪些工具/方法？
R（结果 Result）：最终产生了什么结果？最好有数字支撑。
```

示例（原始）：

```text
“参与某电商平台数据分析项目，负责分析用户行为，帮助优化活动方案。”
```

拆分之后：

```text
S：某电商平台希望提升活动转化率。
T：我负责对活动期间的用户行为数据进行分析。
A：使用 Excel/数据透视表，对 1 万+ 条点击与下单数据进行清洗和分组统计，分析不同入口、不同人群的转化率差异。
R：建议将资源重点投入高转化人群，对低转化入口进行优化测试，最终活动整体转化率提升约 12%。
```

#### 3.3 把 STAR + 数据合并成简历语句

将上述四点压缩成 2–3 行简历条目，注意保留“行动 + 结果 + 数字”：

示例（优化后）：

```text
“负责某电商平台活动数据分析，使用 Excel 数据透视表清洗并分析 1 万+ 条用户行为数据；
通过对不同入口和人群的转化率分组统计，识别出高转化人群特征与低效入口；
提出优化建议后，活动整体转化率提升约 12%。”
```

你可以对 2–3 条重点经历都做类似的处理，并替换进你的简历中。

> 不知道怎么写时，可以把“原始描述 + STAR 四行”给 AI，让它帮你合成一段“STAR+数据风格”的简历语句，再自行修改。

### 步骤四：用 AI 辅助做针对性简历优化（约20分钟）

#### 4.1 让 AI 做一次“体检报告”

复制你的简历全文给 AI，使用类似提示词：

```text
“请帮我分析这份简历：
[粘贴你的简历]

请从以下几个方面给出 1–10 分的评分，并分别给出具体修改建议：
1）结构清晰度；
2）与【数据分析/运营等】岗位的匹配度；
3）数据量化程度；
4）关键词覆盖和针对性；
5）整体表达简洁性。”
```

根据 AI 的反馈，挑几条最关键的建议进行修改，例如：
- 补充缺失的核心技能；  
- 调整经历顺序（把最相关的放在前面）；  
- 删除重复或价值不高的内容；  
- 合并零碎经历，突出亮点。  

#### 4.2 针对具体岗位进行“定向优化”

在完成“通用版”优化后，可以选一个具体 JD，再进行针对性调整：

```text
“我要投递【某某公司】的【数据分析师】岗位，JD 如下：
[粘贴 JD]

下面是我当前修改后的简历：
[粘贴简历]

请帮我：
1）指出简历中与 JD 高度匹配的部分；
2）指出缺失或需要加强的部分；
3）给出 3–5 条具体修改建议，使简历更匹配这个岗位。”
```

根据 AI 的建议，你可以稍微修改一些用词、增加/删减部分经历，使这份简历**更加贴近该岗位的要求**。

如有时间和意愿，可以为第二个岗位方向再做一个“改动不大但重点不同”的简历版本（多版本简历是加分项）。

## ❓ 常见问题（FAQ）

**Q1：我没有正式实习经历，简历上没什么可写的怎么办？**  
**A：** 课程项目也是“项目经历”，尤其是你在本微专业中做过的 4 个项目，都可以整理成简历上的条目。关键是：突出你具体做了什么、用了哪些工具、产生了什么结果。  

**Q2：我觉得自己没什么“结果”，不太好意思写数据？**  
**A：** 结果不一定都是“提升多少万营业额”，也可以是：分析了多少条数据、帮助老师/同学解决了什么问题、在班级/比赛中的排名等。实事求是即可，切忌夸大。  

**Q3：使用 AI 写简历会不会显得“很假”？**  
**A：** AI 可以帮你润色和结构化表达，但**内容必须来自你自己真实的经历**。你可以把 AI 输出当草稿，自己再修改成更符合你语气的表达。面试中能不能讲清楚，是检验“真伪”的关键。  

**Q4：需要准备很多版本简历吗？**  
**A：** 初期不需要太多。建议先有一个“通用的数据分析/运营方向版”，如果你有两个明显不同的目标岗位（比如“数据分析师”和“新媒体运营”），再额外做 1 个重点不同的版本即可。

## 💡 学习提示

- **简历是你和 HR 的第一次对话**：内容要真实、数据要清晰、排版要专业。  
- **优先把“相关度最高”的东西放在前面**：比如和数据分析相关的项目和技能，不要藏在后半页。  
- **不要害怕修改很多遍**：优秀的简历往往是反复修改、在投递和面试反馈中持续打磨出来的。  
- **善用 AI，但不要完全依赖 AI**：让它做你的“写作助理”和“评审老师”，最终版本要经过自己的判断。
