## 🧭 操作总体思路

本任务要解决三个问题：

1. **市场要什么：** 你选择的岗位方向最常要求哪些硬技能和软技能？  
2. **我有什么：** 在这些“市场需要”的技能中，你目前掌握到什么程度？  
3. **差在哪儿、怎么补：** 哪些是必须补的“硬要求”，哪些是可以慢慢补的“加分项”？你打算如何在接下来几周/几个月里提升？  

你会通过“从 JD 中提取技能关键词 → 统计词频 → 对自己打分 → 制定计划”来完成这一轮分析。

## 📋 操作步骤

### 步骤一：搭建技能分析框架（约15分钟）

#### 1.1 理解技能分类

在纸上或 Word 中先简单划分两大类：

- **硬技能（Hard Skills）**  
  - 工具：Excel、SQL、Python、Tableau、Power BI 等  
  - 专业知识：统计学基础、数据分析方法、电商知识等  
  - 方法论：A/B 测试、用户分群、漏斗分析等  

- **软技能（Soft Skills）**  
  - 沟通表达能力  
  - 逻辑思维 / 结构化思考  
  - 学习能力  
  - 团队协作 / 抗压能力 等  

你可以先抄一份这两个列表，后面用来给技能打标签。

#### 1.2 明确分析目标

用一两句话写清楚本任务的个人目标，例如：

```text
本次技能需求分析的目标：
- 明确“数据分析师 / 电商运营（任选你的方向）”岗位最核心的 5–10 个技能；
- 看清自己在哪些核心技能上差距最大；
- 制定一个未来 4–8 周的技能提升小计划。
```

### 步骤二：从 JD 中提取技能关键词并统计词频（约30分钟）

建议在 Excel 中创建 `Sheet1：JD技能提取` 和 `Sheet2：技能词频统计`。

#### 2.1 收集 JD 文本（如果任务13 尚未整理）

从任务13 的岗位数据中选定 1–2 个重点方向（例如：数据分析师 + 电商运营），对每个方向：

- 选 10–20 条代表性的岗位；  
- 将 JD（岗位描述、岗位要求、任职资格部分）复制出来，集中到一个文档或 Excel 的一列中；  

> 如果你已在任务13 中保存了 JD 文本，这一步可以直接复用。

#### 2.2 使用 AI 帮你提取技能关键词（推荐）

选定一个方向（例如“数据分析师”），把若干 JD 合并成一段文本，粘贴给 AI，使用类似提示词：

```text
“以下是多个【数据分析师】岗位的 JD 内容，请帮我：
1）提取所有出现的技能相关关键词（包括工具、方法、知识、软技能）；
2）合并同类项（如 Excel 高级 / Excel 熟练 归为 Excel）；
3）按出现频率从高到低排序；
4）帮我标记每个技能属于【硬技能】还是【软技能】。”

JD 内容：
[粘贴多个 JD]
```

将 AI 输出的结果整理到 `Sheet2：技能词频统计` 中，例如：

| 技能         | 出现次数 | 出现率估计 | 分类   |
|--------------|----------|------------|--------|
| Excel        | 18       | 90%        | 硬技能 |
| SQL          | 15       | 75%        | 硬技能 |
| Python       | 12       | 60%        | 硬技能 |
| 统计学基础   | 10       | 50%        | 硬技能 |
| 数据可视化   | 8        | 40%        | 硬技能 |
| 沟通表达能力 | 12       | 60%        | 软技能 |

> 出现率可以粗略估算：比如 20 条 JD 中有 18 条提到 Excel，就记成 90%。  

#### 2.3 手动微调与分类

你可以根据自己的理解做一点微调：

- 合并语义相近的技能（如“Excel 高级函数 / Excel 透视表”统一为 “Excel 高级应用” 或直接“Excel”）  
- 将所有技能标记为：**必备 / 重要 / 加分** 三类（可按出现率粗分）：  
  - 出现率 > 70%：必备技能  
  - 40%–70%：重要技能  
  - <40%：加分技能  

在表格中新增一列“重要性”：

| 技能 | 出现次数 | 出现率估计 | 分类   | 重要性 |
|------|----------|------------|--------|--------|
| Excel | 18       | 90%        | 硬技能 | 必备   |
| SQL   | 15       | 75%        | 硬技能 | 必备   |
| Python | 12      | 60%        | 硬技能 | 重要   |

### 步骤三：自我技能评估与匹配度计算（约20分钟）

在 Excel 中新增 `Sheet3：自我评估表` 和 `Sheet4：匹配度计算`。

#### 3.1 给自己的技能打分

在 `Sheet3：自我评估表` 中，针对上一节整理出的主要技能（尤其是“必备 + 重要”），对自己进行 0–5 分自评分：

推荐评分标准：

```text
5 分：精通（可以教别人，有项目/实战经验）
4 分：熟练（能独立完成复杂任务）
3 分：掌握（能完成常规任务）
2 分：了解（用过，但需要查资料）
1 分：听说（只知道名词）
0 分：不了解
```

在表格中记录：

| 技能 | 重要性 | 我的评分（0–5） | 简要说明 |
|------|--------|------------------|----------|
| Excel | 必备   | 4                | 课程与项目中大量使用 |
| SQL   | 必备   | 2                | 只写过简单查询 |
| Python | 重要  | 1                | 只学过基础语法 |

#### 3.2 计算技能匹配度（简单版）

可以先做一个“简单匹配度”：  

```text
匹配度 = 达标技能数 / 核心技能数 × 100%
（达标可以定义为评分 ≥ 3 分）
```

例如：
- 核心技能（必备 + 重要）总共有 10 个；  
- 其中你有 7 个技能评分 ≥ 3 分；  
- 那么匹配度 ≈ 70%。  

可以在 Excel 中用 COUNTIF 和简单公式实现。

#### 3.3 计算“加权匹配度”（可选，作为进阶/加分）

如果你想算得更细一点，可以给不同重要性赋予不同权重，例如：

- 必备技能权重：3  
- 重要技能权重：2  
- 加分技能权重：1  

然后用类似公式估算：

```text
加权得分 = Σ(单个技能得分 × 重要性权重) / Σ(满分5分 × 重要性权重)
```

这样可以更好地体现“必备技能”的重要性。

### 步骤四：制定技能提升计划（约25分钟）

#### 4.1 找出“必须优先补”的技能

根据前两步的结果，列出：

- **必备技能中，评分 ≤ 2 分的技能**（这是你当前的“短板核心”）  
- **重要技能中，评分 ≤ 2 分但你又很感兴趣的技能**  

可以整理成一张“差距技能清单”：

| 技能 | 重要性 | 当前评分 | 是否优先补？ | 备注 |
|------|--------|----------|--------------|------|
| SQL  | 必备   | 2        | 是           | 简历与JD都要求 |
| Python | 重要 | 1        | 是           | 未来方向相关   |
| 数据可视化 | 重要 | 2    | 视时间再定   |      |

#### 4.2 为每个优先技能制定学习计划

建议先选择 2–3 个最关键的技能，写出相对详细的计划。可以参考教案中的模板，在 Word 或 Excel 中填写：

```text
【技能】：SQL
【当前水平】：2 分（了解，写过简单查询）
【目标水平】：4 分（熟练，能独立写复杂查询和分析）
【学习周期】：4 周

【每周计划】：
- 第 1 周：熟悉基础语法（SELECT、WHERE、GROUP BY、ORDER BY）
- 第 2 周：掌握多表关联（JOIN）、子查询等进阶用法
- 第 3 周：通过在线平台刷题（例如 LeetCode / 牛客 SQL 题 30–50 道）
- 第 4 周：使用真实数据做一个小项目（例如分析一张订单表/用户表）

【学习资源】：
- 网课/视频：________
- 书籍/文档：________
- 练习平台：________

【验证方式】：
- 至少完成 ___ 道 SQL 练习题；
- 能独立完成 1 个简单的数据分析小项目。
```

重复上述步骤，为 2–3 个技能分别写计划。

#### 4.3 借助 AI 制定与优化计划

当你有了一个“草稿版计划”之后，可以复制给 AI，请它帮你检查和补充，例如：

```text
“我想在 4 周内把【SQL】从【了解】提升到【熟练】，以下是我自己的学习计划：
[粘贴你的计划]

请你从合理性、难度、资源推荐三个方面帮我优化：
1）调整每周的学习重点和时间安排；
2）推荐适合的免费/低成本学习资源；
3）补充一些实战练习建议。”
```

## ❓ 常见问题（FAQ）

**Q1：我的技能都很基础，自评分数都不高，怎么办？**  
**A：** 分数低并不是问题，关键是你现在知道“差在哪”，并且有清晰的补齐计划。老师和用人单位更看重的是**清晰的自我认知 + 明确的成长路径**，而不是一开始就“完美”。  

**Q2：从 JD 里看起来要会的东西太多了，学不过来？**  
**A：** 这正是需要“分类和排序”的原因。通过词频和重要性分类，你会发现真正“高频必备”的技能其实就那几项，先把它们学扎实，再逐步扩展就好。  

**Q3：自我评分会不会太主观、不准确？**  
**A：** 可以找同学/老师一起互评，也可以让 AI 帮你设计一些“自测小题目”，通过能否完成这些题目来判断自己的水平。最重要的是保持诚实，千万不要“高估自己”写进简历却撑不住面试追问。  

**Q4：学习计划写得很漂亮，但我怕坚持不下来？**  
**A：** 你可以在任务16 的“求职跟踪系统”中，顺便加入一个“技能学习打卡”区域，把计划拆成更小的动作（每天/每周），用数据记录自己的坚持情况，这样会更容易坚持。

## 💡 学习提示

- **岗位要求不是“吓人”，而是“指路牌”**：它告诉你行业真正看重什么技能，少走弯路。  
- **先把“少数关键技能”提升到及格线以上**，再去追求“更多的新技能”。  
- **善用 AI 做你的“学习教练”**：让它帮你拆解学习路径、推荐资源、出练习题。  
- **把计划写细一点，越具体越容易执行**：比如“每天刷 3 道 SQL 题”要比“多练习 SQL”更容易做到。
