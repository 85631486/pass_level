### 步骤一：什么是数据故事（12分钟）

#### 1.1 数据故事的定义

**定义：**
```
数据故事 = 数据 + 可视化 + 叙事

数据：事实和证据
可视化：视觉呈现
叙事：引导观众理解的结构
```

**对比：**

**只有数据：**
```
"我们公司2023年销售额1000万，2024年1500万"
效果：干巴巴的数字
```

**只有可视化：**
```
一个柱状图，没有标题，没有解释
效果：看不懂要表达什么
```

**完整的数据故事：**
```
标题：销售额增长50%，移动端贡献最大
开头：2023年销售额1000万
中间：2024年增至1500万，其中移动端增长80%
结尾：应加大移动端投入
配图：折线图+柱状图
效果：清晰、有说服力
```

#### 1.2 优秀数据故事案例

**案例A：《纽约时报》"枪支暴力"数据新闻**
```
主题：美国枪支暴力问题严重性
开头：引出问题背景
中间：用多个图表展示数据（时间趋势、地域分布、对比其他国家）
结尾：呼吁政策改变
效果：震撼、引人深思
```

**案例B：Hans Rosling的TED演讲**
```
主题：世界卫生和经济发展
方式：动态气泡图展示各国变化
叙事：像讲故事一样解读数据
效果：枯燥数据变得生动有趣
```

**案例C：学生个人消费分析**
```
主题：我的大学消费习惯变化
开头：大一时的消费情况
中间：各年级消费变化，原因分析
结尾：理财建议和未来规划
效果：贴近生活，有共鸣
```

### 步骤三：选择故事主题（15分钟）

#### 3.1 主题选择原则

**原则1：有数据支撑**
```
✓ 你有相关数据
✗ 只是一个想法，没有数据
```

**原则2：有明确的洞察**
```
✓ 数据揭示了有趣的发现
✗ 数据平淡无奇，没什么可说的
```

**原则3：观众感兴趣**
```
✓ 与观众相关或引发思考
✗ 过于小众或无聊
```

**原则4：可行性强**
```
✓ 在时间和能力范围内可完成
✗ 数据太复杂或太多
```

#### 3.2 主题示例

**个人类：**
```
- 我的大学消费变迁
- 我的学习时间分配
- 我的健康数据（体重、运动）
- 我的阅读习惯分析
```

**班级类：**
```
- 班级学习情况分析
- 同学通勤方式对比
- 期末成绩分布与影响因素
- 课外活动参与度调查
```

**社会类：**
```
- 00后消费观念研究
- 大学生兼职情况调查
- 城市生活成本对比
- 疫情对生活方式的影响
```

**商业类：**
```
- 某产品市场分析
- 竞品对比研究
- 用户行为洞察
- 行业趋势预测
```

### 步骤四：规划故事结构（20分钟）

#### 4.1 三幕式结构

**结构框架：**
```
第一幕：开头（Setup）
- 背景介绍
- 提出问题
- 引起兴趣

第二幕：中间（Confrontation）
- 展示数据
- 深入分析
- 揭示发现

第三幕：结尾（Resolution）
- 得出结论
- 提供建议
- 呼吁行动（可选）
```

#### 4.2 故事线设计技巧

**技巧1：设置悬念**
```
开头抛出问题，中间逐步揭晓
例："为什么我的支出突然大幅下降？"
```

**技巧2：对比反差**
```
预期vs实际
自己vs他人
过去vs现在
```

**技巧3：情感连接**
```
用个人经历引发共鸣
例："每次看到外卖账单都心疼，直到我开始自己做饭..."
```

**技巧4：数据亮点**
```
突出最有趣的发现
例："没想到我在奶茶上花了5000元！"
```

#### 4.3 学生练习

**任务：规划你的故事线**

**填写故事规划表：**

### 步骤五：绘制故事板（20分钟）

#### 5.1 什么是故事板

**定义：**
```
故事板（Storyboard）：
用草图展示故事的流程和关键画面
类似电影分镜头脚本
```

**作用：**
```
- 可视化故事结构
- 规划图表内容和位置
- 检查逻辑连贯性
- 便于团队沟通
```

**格式：**
```
一系列方框，每个方框代表一个"画面"
画面 = 一张图表 + 文字说明
```

#### 5.2 故事板模板

**基础模板：**
```
┌─────────────────┐
│  画面1：开头      │
│                 │
│  [草图：标题页]   │
│  文字：背景介绍   │
└─────────────────┘

┌─────────────────┐
│  画面2：问题      │
│                 │
│  [草图：关键问题] │
│  文字：提出问题   │
└─────────────────┘

┌─────────────────┐
│  画面3：发现1     │
│                 │
│  [草图：柱状图]   │
│  文字：XX趋势     │
└─────────────────┘

...

┌─────────────────┐
│  画面N：结尾      │
│                 │
│  [草图：总结页]   │
│  文字：结论建议   │
└─────────────────┘
```

#### 5.3 绘制示范

**强调：不需要画得精美，草图即可！**


#### 6.2 图表清单模板

**模板：**
```
| 序号 | 图表用途 | 图表类型 | 数据来源 | 优先级 |
|------|---------|---------|---------|--------|
| 1 | 年度总消费 | 柱状图 | 支付宝账单 | 高 |
| 2 | 支出类别 | 饼图 | 汇总数据 | 高 |
| 3 | 月度趋势 | 折线图 | 月账单 | 高 |
| 4 | 外卖vs食堂 | 组合图 | 分类数据 | 中 |
| 5 | 总结页 | 信息图 | - | 中 |
```

**关键字段：**
```
- 序号：按故事顺序编号
- 图表用途：在故事中的作用
- 图表类型：柱状图、折线图等
- 数据来源：数据从哪来
- 优先级：高/中/低（时间不够可削减低优先级）
```

### 常见问题FAQ

**Q1: 我的数据很枯燥，怎么讲出故事？**
A: 故事来自洞察，不是数据本身：
   - 找对比（预期vs实际）
   - 找变化（时间趋势）
   - 找关系（相关性）
   - 加入个人经历和感受

**Q2: 数据故事一定要有结论吗？**
A: 最好有，但也可以：
   - 提出问题，引发思考
   - 展示发现，让观众自己判断
   - 承认不确定性，提出假设

**Q3: 故事板一定要手绘吗？**
A: 不一定：
   - 手绘：最快，自由
   - PPT：整洁，易修改
   - 专业工具（Figma）：精美
   - 选择你最顺手的方式
