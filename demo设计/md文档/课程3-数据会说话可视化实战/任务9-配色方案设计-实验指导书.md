### 步骤三：配色类型讲解（25分钟）

#### 3.1 配色的三种类型

**类型1：定性配色（Qualitative）**
```
用途：区分不同类别，无大小关系
特点：颜色差异明显，没有顺序
示例：产品A=蓝色，产品B=橙色，产品C=绿色
```

**类型2：顺序配色（Sequential）**
```
用途：编码数值大小，有顺序关系
特点：单一色调，深浅渐变
示例：浅蓝→深蓝 代表 低值→高值
```

**类型3：发散配色（Diverging）**
```
用途：展示偏离中心值的程度
特点：两种色调，从中间向两端发散
示例：蓝色←白色→红色 代表 负值←0→正值
```

#### 3.2 定性配色详解

**何时使用：**
- 柱状图的不同类别
- 折线图的不同系列
- 饼图的不同部分

**配色原则：**

**原则1：颜色易于区分**
```
✓ 好：蓝色、橙色、绿色、紫色
✗ 差：浅蓝、中蓝、深蓝、更深蓝
```

**原则2：数量控制在5-6种**
```
✓ 3-5种：最佳
⚠ 6-8种：可接受
✗ 10+种：太多，难以区分
```

**原则3：避免红绿组合（色盲友好）**
```
✗ 红色vs绿色：红绿色盲无法区分
✓ 蓝色vs橙色：色盲友好
✓ 蓝色vs黄色：色盲友好
```

**推荐定性配色方案：**
```
方案1（ColorBrewer Set2）：
#66C2A5 #FC8D62 #8DA0CB #E78AC3 #A6D854

方案2（Tableau 10）：
#4E79A7 #F28E2B #E15759 #76B7B2 #59A14F

方案3（蓝橙对比）：
#1F77B4 #FF7F0E #2CA02C #D62728 #9467BD
```

#### 3.3 顺序配色详解

**何时使用：**
- 热力图
- 地图（人口密度、温度等）
- 条件格式色阶
- 单一维度的数值编码

**配色原则：**

**原则1：使用单一色调**
```
蓝色系：浅蓝→深蓝
绿色系：浅绿→深绿
灰色系：浅灰→深灰
```

**原则2：确保足够的对比度**
```
最浅色：接近白色，可读
最深色：足够深，可读
中间：平滑过渡
```

**原则3：方向一致**
```
惯例：深色=高值，浅色=低值
除非有特殊原因，否则遵循惯例
```

**推荐顺序配色方案：**
```
蓝色（Blues）：
#EFF3FF #C6DBEF #9ECAE1 #6BAED6 #3182BD #08519C

绿色（Greens）：
#EDF8E9 #C7E9C0 #A1D99B #74C476 #31A354 #006D2C

橙色（Oranges）：
#FEE6CE #FDD0A2 #FDAE6B #FD8D3C #E6550D #A63603
```

#### 3.4 发散配色详解

**何时使用：**
- 有明确中心值的数据（如0、平均值）
- 展示正负偏差
- 对比两种相反的状态

**配色原则：**

**原则1：选择对比明显的两种色调**
```
常用组合：
蓝-红：蓝色（负/冷）←→红色（正/热）
绿-紫：绿色（低）←→紫色（高）
棕-青：棕色（干）←→青色（湿）
```

**原则2：中间使用白色或浅色**
```
蓝色←浅灰/白色→红色
让中心值（如0）清晰可辨
```

**推荐发散配色方案：**
```
红蓝（RdBu）：
#B2182B #D6604D #F4A582 #FDDBC7 #D1E5F0 #92C5DE #4393C3 #2166AC

棕绿（BrBG）：
#8C510A #BF812D #DFC27D #F6E8C3 #C7EAE5 #80CDC1 #35978F #01665E
```

### 步骤六：配色常见错误（8分钟）

#### 6.1 错误1：使用彩虹配色

**问题：**
```
7个类别，用彩虹7色
红、橙、黄、绿、青、蓝、紫
```

**为什么不好：**
- 颜色有隐含顺序（光谱），但数据没有顺序
- 过于花哨，缺乏专业感
- 有些颜色对比度不够

**替代方案：**
```
使用专业定性配色，如ColorBrewer Set2
或单色系+强调色
```

#### 6.2 错误2：红绿对比（色盲问题）

**问题：**
```
盈利=绿色
亏损=红色
```

**为什么有问题：**
- 约8%男性有红绿色盲
- 他们看到的红绿是同样的棕色

**替代方案：**
```
盈利=蓝色
亏损=橙色
或添加图标/纹理辅助区分
```

#### 6.3 错误3：对比度不足

**问题：**
```
浅黄色背景+白色文字
浅灰色线条在白色背景上
```

**为什么不好：**
- 难以阅读
- 投影时完全看不见

**解决方法：**
```
使用对比度检查工具
确保文字和背景对比度至少4.5:1
```

#### 6.4 错误4：颜色过多

**问题：**
```
10个类别，10种不同颜色
```

**为什么不好：**
- 难以记忆和区分
- 视觉混乱

**解决方法：**
```
合并小类别为"其他"
或使用分组：同一组用同色系不同深浅
```

## 📚 附录

### 常见问题FAQ

**Q1: 怎么知道我的配色是否色盲友好？**
A: 使用色盲模拟工具，如：
   - Coblis（在线工具）
   - ColorOracle（桌面软件）
   - Adobe Color的色盲模拟功能

**Q2: 品牌色和数据可视化配色冲突怎么办？**
A: 以品牌色为主色，搭配中性色（灰色）和一个对比色。不必所有图表都用品牌全色系。

**Q3: 深色背景（如深色PPT）的图表怎么配色？**
A: 反转亮度：
   - 用浅色代替深色
   - 确保文字是白色或浅色
   - 测试对比度

**Q4: Excel图表能用渐变色吗？**
A: 可以，但要慎用：
   - 适合单一系列的装饰
   - 不适合多系列对比
   - 可能影响可读性

### 配色速查表

**场景：对比不同类别（定性）**
→ 推荐：ColorBrewer Set2、Tableau 10
→ 颜色数：3-6种

**场景：展示数值大小（顺序）**
→ 推荐：单色系渐变（Blues、Greens）
→ 方向：浅→深

**场景：展示正负偏差（发散）**
→ 推荐：蓝-红、棕-青
→ 中心：白色或浅灰

**场景：突出重点**
→ 方法：重点=鲜艳色，其他=灰色
→ 示例：蓝色+5个灰色

## 🆘 卡住了？试试这些
- **HEX 码找不到入口**：在“设置数据系列格式→填充→更多颜色→自定义”中输入 #RRGGBB；或在主题颜色对话框里设置。
- **颜色被自动覆盖**：把图表样式改成“样式1”，再手动填色；或者应用自定义主题后再微调。
- **色盲友好不知道怎么测**：把图表截图上传到 Coblis/ColorOracle 模拟，或问 AI “帮我检查这个配色是否色盲安全”。
- **配色和品牌色冲突**：以品牌主色为强调色，再搭配灰度或同色系深浅即可。
- **Excel 导出的图片颜色变淡**：勾选“另存为图片”时选择 PNG，并保持原大小；在 PPT 中复制粘贴时不要拉伸比例。
