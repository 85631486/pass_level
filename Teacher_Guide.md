# 过关斩将教学平台 - 教师端功能指南

本指南旨在为教育工作者提供「过关斩将」教学平台的深度操作指引。平台通过可视化与智能化的手段，协助教师高效完成从课程大纲到交互式闯关地图的转化。

## 1. 核心模块体系

教师端的操作逻辑围绕课程的生命周期展开，主要由以下三个核心支柱构成。

| 模块名称 | 核心功能 | 应用场景 |
| :--- | :--- | :--- |
| **课程地图设计** | 鱼骨图可视化编辑、AI 自动建模 | 课程结构规划、知识体系可视化 |
| **关卡内容编辑** | 任务配置、卡片设计、环节步骤编排 | 教学内容填充、交互式学习设计 |
| **闯关考题管理** | 题库建设、通过标准设定 | 学习效果评估、质量控制 |

---

## 2. 课程地图设计 (LevelMapEditor)

课程地图是学生学习路径的视觉呈现，采用**鱼骨图（Fishbone Diagram）**拓扑结构，能够清晰地展示知识的演进脉络。

### 2.1 可视化交互逻辑
教师可以通过直观的拖拽操作来构建地图。**根节点 (Root)** 代表章节核心，位于鱼骨的最右端；**模块节点 (Branch)** 承载主要的教学单元，直接挂载在主骨架上；**任务节点 (Sub)** 则是最细粒度的学习单元。系统内置的布局算法会自动处理节点间的连线与锚点位置，确保图谱的整洁与逻辑一致性。

### 2.2 AI 智能建模
平台集成了先进的大语言模型，支持从原始教学大纲一键生成地图。教师只需提供文本形式的大纲，AI 即可自动识别知识层级并完成鱼骨图布局。该功能已针对教育场景进行了深度优化，能够自动过滤非教学信息，确保生成的节点完全聚焦于核心知识点。

---

## 3. 关卡内容编辑 (LevelEditorEnhanced)

在宏观地图构建完成后，教师需要对每个任务节点进行微观内容的填充。

### 3.1 学习资源配置
教师可以为每个关卡配置**知识卡片**与**技能卡片**。这些卡片支持 Markdown 语法，允许嵌入富文本、代码块及多媒体资源，为学生提供沉浸式的理论学习环境。

### 3.2 交互式环节设计
通过**环节与步骤 (Phases & Steps)** 模块，教师可以将复杂的任务拆解为多个可操作的阶段。平台提供的可视化编辑器支持组件化布局，教师可以像搭积木一样设计交互页面，定义学生的点击、输入及反馈逻辑。

---

## 4. 教学质量保障

为了确保教学目标的达成，平台提供了完善的考核与发布机制。

| 功能项 | 描述 | 关键作用 |
| :--- | :--- | :--- |
| **考题配置** | 支持单选、多选、判断等多种题型 | 检验学生对知识点的掌握程度 |
| **通过标准** | 设定关卡解锁的最低正确率要求 | 实施闯关式学习，确保学习质量 |
| **预览模式** | 模拟学生视角进行全流程体验 | 教师自检，优化教学体验 |
| **发布管理** | 一键切换草稿与发布状态 | 灵活控制教学进度，支持版本迭代 |

---

## 5. 维护与扩展建议

教师在维护课程时，应优先利用 AI 辅助功能进行结构化调整。如需自定义复杂的交互逻辑，建议参考 `VisualCourseEditor` 组件的扩展接口。所有的地图配置均以标准 JSON 格式存储，便于跨平台迁移与备份。
