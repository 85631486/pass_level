## 平台定位与目标

- **项目名称**：过关斩将-游戏教学平台  
- **服务对象**：大学老师与学生，聚焦 00 后学习人群  
- **总体目标**：将课程拆解成“关卡任务”，用即时奖励与闯关机制提升学习兴趣、完成率与课堂互动度。

## 角色与核心需求

- **教师**  
  - 创建课程与教学任务，配置任务资源（视频/作业指导书/过程提交要求/过关考题）。  
  - 课堂前选择任务发布，掌控教学节奏与难度。  
  - 查看学生任务完成情况与考题通过率，进行针对性反馈。
- **学生**  
  - 登录后查看教师当前发布的任务，按步骤完成、提交材料并闯关考试。  
  - 获得即时奖励、等级/称号/排名，保持学习粘性。  
  - 查看历史闯关记录与错题，制定复习计划。
- **教务/运营（可选）**  
  - 维护题库模板、审核课程资源、监控运行数据与教学质量。

## 系统模块概览

| 模块 | 子功能 | 说明 |
| --- | --- | --- |
| 课程管理（教师端） | 课程档案、班级绑定、任务库 | 支持复制课程模板与批量导入任务 |
| 任务编排 | 任务基本信息、资源上传、提交要求配置、过关考题配置 | 每个任务包含：资源（视频/指导书）、任务步骤/提交项、考题（多题型） |
| 发布与通知 | 上课前一键发布、定时发布、消息推送 | 与班级群/短信/站内信联动 |
| 学生任务中心 | 待办任务、资源学习、过程提交、过关答题、复盘 | 支持移动端自适应与断点续做 |
| 题库与判题 | 题型模板（单选、多选、判断、填空、简答、编程等）、自动判分、主观题批阅 | 支持导入导出与 AI 纠错辅助（可拓展） |
| 激励系统 | 经验值、积分、徽章、关卡星级、排行榜、成就故事线 | 奖励规则与任务难度关联，支持教师自定义 |
| 数据分析 | 学生画像、任务完成漏斗、知识点掌握度、课堂活跃度、异常预警 | 提供图表与原始数据导出 |
| 系统管理 | 用户/角色、权限、资源审核、题库模板维护 | 支持 SSO/校园统一认证对接 |

## 典型业务流程

1. **教师建课**：新建课程 → 绑定班级 → 从模板或历史课程复制任务。  
2. **设计任务**：  
   - 录入任务简介与目标。  
   - 上传视频或作业指导书。  
   - 设定任务完成的过程提交项：如实验截图、报告 PDF、代码仓库链接。  
   - 关联过关考题（题库选题或新增题目），配置判分规则与通过条件。  
3. **发布任务**：上课前从任务列表中选择要发布的关卡，可实时或定时推送。  
4. **学生闯关**：  
   - 查看任务 → 学习资源（视频/文档） → 按步骤提交材料。  
   - 进入过关考题作答，系统即时判分；未通过可查看提示并重试（限定次数）。  
   - 完成后获得经验值/积分/徽章，并解锁下一任务。  
5. **结果追踪**：教师在课堂仪表盘查看完成率、平均用时、知识点薄弱项，课堂上即时点名辅导。  
6. **课堂回顾**：教师批改主观题、生成成绩报告；学生查看错题本与过程材料反馈。

## 激励与游戏化设计

- **即时反馈**：任务提交后实时评分与进度条；过关页面展示动画与奖励。  
- **多层奖励**：  
  - 任务级：经验值、金币、星级评价。  
  - 章节级：宝箱（随机奖励）、称号。  
  - 学期级：排行榜（个人/团队）、成就路线图。  
- **社交互动**：组队闯关、互助提示、点赞优秀任务文档。  
- **运营活动**：节日主题赛、限时挑战、副本任务（跨课程合作）。

## 数据与内容结构（概要）

- **课程**：`课程ID`、名称、学期、班级列表、模板标记。  
- **任务**：`任务ID`、课程关联、资源集合（视频、文档）、过程提交项（类型、数量、检查点）、过关条件。  
- **考题**：题型、难度、知识点标签、判分方式、参考答案、解析。  
- **学生任务记录**：状态、提交物路径、评分、重试次数、奖励发放记录。  
- **激励数据**：经验曲线、积分商城（若有）、徽章配置、排行榜快照。  
- **日志与监控**：行为日志、异常告警（未提交、低分、挂机观看等）。

## 技术与实现建议（可迭代）

- **前端**：React/Vue + TypeScript，支持响应式布局；H5 优先，后续可封装小程序/APP。  
- **后端**：Spring Boot / NestJS / .NET Core 等，模块化设计（课程、任务、题库、激励、统计）。  
- **数据库**：关系型数据库（MySQL/PostgreSQL）存储主业务；Redis 做缓存与排行榜；对象存储存放音视频与提交材料。  
- **音视频**：对接云点播/课堂录制服务，提供试看与倍速。  
- **判题与评分**：客观题即时判卷；主观题批阅支持富文本批注；可扩展 AI 辅助批改。  
- **监控与分析**：埋点数据进入 BI（如 ClickHouse + Superset）或使用日志平台。  
- **集成**：支持学校统一账号（SSO）、教务系统的课表与成绩同步。

## 项目推进建议

1. **原型设计**：快速产出教师端/学生端原型，验证任务编排与闯关体验。  
2. **模块迭代顺序**：任务闭环（课程-任务-考题-过关） → 激励系统 → 数据分析 → 扩展功能。  
3. **试点与反馈**：选 1-2 门课程试运行，收集学生对难度与奖励的反馈，调优任务颗粒度。  
4. **沉淀模板**：将优秀课程沉淀为任务模板、题库模板，降低老师建课成本。  
5. **扩展计划**：接入 AI 题目生成、学习画像推荐、跨校联赛等高阶功能。
